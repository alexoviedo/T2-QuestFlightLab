# Project Instructions

- Work only from a clean, understood repository state. Inspect the branch, worktrees, stashes, history, remotes, and uncommitted changes before implementation.
- Preserve intentional local work. Never reset, clean, overwrite, or discard it blindly.
- Do not commit Unity `Library`, `Temp`, `Obj`, `Logs`, `Builds`, APKs, raw captures, package caches, raw source archives, or large raw geospatial downloads.
- Do not replace or remodel the imported aircraft/cockpit model unless Alex explicitly requests it.
- Documentation alone is not implementation progress.
- Every visual change requires inspected before/after Visual QA output.
- Every physics change requires scenario evidence.
- Never modify the tracked XR camera transform directly to fake seat motion.
- Never use the headset/camera, XR Origin, or pilot seat as the aircraft rotation pivot.
- Keep the mesh/environment fallback available.
- Do not use Google Maps, Google Earth, or Street View-derived content as committed assets.
- Use only free/open/public-domain/self-generated project assets, and record each source and license.
- Do not claim FAA approval, training credit, final C172 fidelity, production photorealism, broad compatibility, or final Quest performance without evidence.
- Run the standard scenario suite, PlayMode tests, Visual QA, and Quest APK build before committing.
- Inspect every staged binary and asset before committing.
- Remove temporary worktrees and branches before completion unless explicitly justified.
- Final responses must disclose failures and limitations honestly.
