# Scenic Splat Asset Pipeline

## v0.7 Asset Decision

v0.7 uses a Codex-created procedural scenic splat patch for the playable demo.

This asset is:

- synthetic and project-owned,
- generated by `tools/generate_scenic_splat_patch.py`,
- intended as an opt-in background/scenic layer,
- not a real KBDU scan,
- not a photogrammetry capture,
- not for navigation,
- not part of aircraft, cockpit, UI, colliders, training gates, scoring, or flight dynamics.

The mesh/terrain airport remains the default and reliable playable scene.

## Why Procedural First

The v0.6c Quest runtime gate proved that the current Unity Gaussian splat renderer can load synthetic static/background-style samples up to 100k splats on one Quest 3. It did not prove real airport capture viability, long-session thermal behavior, production photorealism, or a full scenery pipeline.

A procedural asset is the safest next step because it avoids external licensing risk and lets the playable demo test the real runtime integration pattern immediately.

## Generated Content

The generator creates a stylized "real-ish" airfield/foothills patch:

- distant foothill color mass,
- dry grass/field ground patch,
- tree-line and airport-edge masses,
- small hangar/airport-edge color blocks,
- atmospheric distant edge points.

The generated patch is visual only. It is intentionally placed away from the runway collision path and should not obscure cockpit instruments, approach gates, runway markings, or lesson panels.

## Runtime Budgets

v0.7 integrates these opt-in modes:

| Mode | Target splats | Purpose |
| --- | ---: | --- |
| `mesh` | 0 | Default playable mesh/terrain fallback. |
| `scenic_splat_low` | 25,000 | First playable scenic layer. |
| `scenic_splat_medium` | 50,000 | Main scenic patch candidate. |
| `scenic_splat_high` | 100,000 | Optional upper check, only if low/medium stay healthy. |

The older synthetic regression modes remain available:

- `splat_5k`
- `splat_50k`
- `splat_100k`

## Performance Guidance

PlayCanvas/SuperSplat performance guidance is used as a general planning reference: Gaussian count matters because splats are sorted by camera depth, and fill-rate/overdraw is a major rendering cost because overlapping translucent billboard-like splats can multiply fragment work. It also recommends LOD streaming and a global splat budget for larger scenes. See: https://developer.playcanvas.com/user-manual/gaussian-splatting/building/performance/

QuestFlightLab v0.7 therefore keeps the scenic splat patch:

- optional,
- budgeted,
- static,
- trimmed to small scene patches,
- away from near-field cockpit/runway surfaces,
- fallback-safe.

## Future Owned Capture Path

For real-world scenic patches, the preferred path is an Alex-owned capture:

1. Capture a small room, landscape, or airfield-adjacent patch with an owned capture tool such as Scaniverse.
2. Export SPZ or PLY.
3. Convert/import into the Unity Gaussian splat renderer path.
4. Trim aggressively.
5. Test Quest runtime at 25k, 50k, and 100k budgets before increasing scope.

Niantic SPZ is a strong future delivery candidate because the open-source README describes SPZ as a compressed 3D Gaussian splat format and says SPZ files are typically about 10x smaller than equivalent PLY files with minimal visual difference. See: https://github.com/nianticlabs/spz

SPZ is not a blocker for v0.7 because the current Unity integration already imports renderer-compatible PLY into Unity runtime assets.

## External Samples

Meta Spatial SDK Gaussian splat samples remain reference material only unless licensing is explicitly cleared for this Unity/OpenXR project. The v0.7 demo does not import Meta sample assets.

Public sample assets should only be used if the license is clear, the size is small, and the asset can be documented. No public or Meta sample is required for v0.7.

## Limitations

- This is not proof of full-airport splat viability.
- This is not production photorealism.
- This is not a real KBDU capture.
- This is not final Quest performance evidence beyond tested runtime modes.
- This does not change the claim boundary for flight training, FAA approval, or C172 fidelity.
- Mesh fallback remains the default.
